# CLAUDE.md — ТЗ: Закритий Telegram-бот «AI-консультант по продуктах банку» (прототип)

> Цей файл — інструкція для Claude Code. Прочитай повністю перед написанням коду.
> Відповідай і коментуй код українською. Стек — TypeScript/Node.

---

## 1. Що це за проєкт

- **Назва (робоча):** `premium-banking-bot`
- **Суть:** **закритий** Telegram-бот, який відповідає на питання про продукти банку (тарифи, картки, лаунжі, переваги преміум-пакетів).
- **Джерела правди (два рівні):**
  1. **Головне:** база знань `help.sensebank.com.ua` (через локальний RAG) — авторитетне джерело для продуктів і умов банку.
  2. **Додаткове:** живий **веб-пошук, обмежений білим списком офіційних доменів** (Visa, Mastercard, DragonPass/LoungeKey, офіційні сайти аеропортів, НБУ тощо) — для загальної інформації, якої немає в базі банку.
- **Доступ:** приватний — відповідає лише користувачам із allowlist.
- **Технологія відповідей:** RAG + вбудований інструмент `web_search` Claude API з `allowed_domains`.
- **Мета:** робочий прототип/демо для портфоліо та показу банку.

### Чим цей проєкт НЕ є

- НЕ офіційний продукт банку, не підключається до внутрішніх систем.
- Не обробляє персональні дані клієнтів.
- Бренд Sense — лише для внутрішнього демо.

### ⚠️ Важливе обмеження по «зовнішніх сайтах»

- Проіндексувати «всі офіційні сайти світу» **неможливо**. Реалізуємо **живий веб-пошук по фіксованому білому списку доменів** — тільки офіційні джерела, ніякого відкритого вебу.
- Без allowlist доменів зовнішній пошук = SEO-сміття й форуми = галюцинації. Allowlist — обов'язковий.

## 2. Технологічний стек

- **Runtime:** Node.js + TypeScript (звичайний процес, без Next.js/React).
- **Telegram:** **grammY** (TS-first; альтернатива — Telegraf).
- **LLM:** Claude API через `@anthropic-ai/sdk`.
- **Модель:** `claude-sonnet-4-6` для **всього** (єдина модель — простіше й якісніше; підтримує новіший інструмент веб-пошуку з динамічною фільтрацією)
- **Веб-пошук:** вбудований інструмент Claude API.
  - Базова версія `web_search_20250305` (працює широко). Новіші версії (`web_search_20260209`+) з динамічною фільтрацією потребують моделей 4.6+. **Перевір актуальну версію в доках перед інтеграцією.**
  - Обов'язково `allowed_domains` (білий список) + `max_uses` (кеп пошуків на хід, напр. 2–3).
  - ⚠️ Веб-пошук треба **увімкнути в Console** (адмін організації). Доменні обмеження на рівні запиту мають бути в межах налаштувань Console.
- **Ембединги (локальна база):** мультимовна модель через Transformers.js — `@huggingface/transformers` (раніше `@xenova/transformers`; перевір назву/версію).
  - Модель: мультимовна (напр. `intfloat/multilingual-e5-small`), бо база **українською**. Англомовний `all-MiniLM` НЕ використовувати.
- **Векторне сховище:** JSON-файл + cosine similarity в пам'яті.
- **Запуск:** long polling.

## 3. Архітектура (простими словами)

Потік одного повідомлення:

1. Користувач пише боту.
2. **Перевірка доступу:** `user.id` не в allowlist → «бот приватний», стоп (Claude не викликається).
3. Питання → ембединг → пошук у локальній базі Sense (топ-4).
4. Виклик Claude API: передаємо знайдені шматки бази + **прикріплюємо інструмент `web_search`** з `allowed_domains`.
5. Claude вирішує сам:
   - є відповідь у базі Sense → відповідає з неї;
   - немає → шукає на дозволених офіційних доменах;
   - ніде немає → чесне «не знаю».
6. Бот надсилає відповідь + посилання на джерела (і з бази, і з вебу).

```
Повідомлення → [allowlist?] → ні → «бот приватний»
                   │ так
                   ▼
   [ембединг] → [пошук у базі Sense] → топ-4 шматки
                   │
                   ▼
   Claude API (контекст бази + tool: web_search[allowed_domains])
        ├─ є в базі → відповідь з бази
        ├─ нема → пошук по офіційних доменах → відповідь
        └─ ніде нема → «не знаю»
                   │
                   ▼
        відповідь + джерела → Telegram
```

> Щоб контролювати витрати, можна спершу гейтити: викликати веб-пошук лише якщо локальний пошук дав результат нижче порога. Для MVP достатньо `max_uses` + allowlist доменів.

## 4. Функціональні вимоги (MVP)

### Команди / поведінка

- `/start` — привітання + дисклеймер (демо, не офіційний сервіс).
- `/help` — що вміє бот, приклади питань.
- `/myid` — повертає Telegram ID користувача (щоб додати в allowlist).
- Текстове повідомлення (від дозволеного користувача) → відповідь через RAG + (за потреби) веб-пошук.
- Індикатор «друкує…» під час обробки.
- Під відповіддю — список джерел (база Sense та/або офіційні сайти).
- `npm run ingest` — одноразова індексація бази.

### Поза рамками MVP

- Інтеграція з реальними банківськими API.
- Веб-інтерфейс, голос/фото (тільки текст).
- Відкритий веб-пошук (тільки allowlist доменів).

## 5. 🔒 Контроль доступу (закритий бот)

1. **Allowlist.** env `ALLOWED_USER_IDS` — числові Telegram ID через кому. Middleware перевіряє `ctx.from.id` **перед** будь-якою обробкою.
2. Не в списку → «Цей бот приватний. Доступ за запрошенням.» (Claude і веб-пошук не викликаються — нуль витрат).
3. Додавання людей: `/myid` → людина надсилає тобі число → додаєш у env → перезапуск.

### Контроль витрат (легкий, бо бот закритий + є веб-пошук)

- **Жорсткий cap у Console** (`console.anthropic.com`) — ліміт витрат, напр. $5.
- `max_uses` на веб-пошук (2–3 на хід) — щоб один запит не зробив десяток пошуків.
- Веб-пошук тарифікується **окремо** від токенів (порядку $0.01 за пошук — перевір актуальне на сторінці цін). Логуй `server_tool_use.web_search_requests` з відповіді, щоб бачити кількість пошуків.
- Простий per-user rate limit + обмеження довжини вхідного тексту.

## 6. 🛡️ Анти-галюцинаційні правила (НАЙВАЖЛИВІШЕ по суті)

1. **Тільки джерела.** Claude відповідає лише з бази Sense або з дозволених офіційних доменів. Жодних знань «з голови».
2. **«Не знаю» — правильна відповідь.** Нема в джерелах → чесно кажемо, направляємо в застосунок/підтримку.
3. **Пріоритет бази банку.** Для умов саме продуктів банку (тарифи, ліміти, переваги карток) база Sense > зовнішні сайти. Зовнішні домени — для загальних правил платіжних систем і фактів про аеропорти.
4. **Загальне ≠ персональне (КЛЮЧОВЕ).** «Visa Signature дає лаунж» (із visa.com) — це загальне правило, а не гарантія для конкретної картки клієнта. У таких відповідях завжди додавати: «перевірте умови саме вашої картки в застосунку банку».
5. **Жодних вигаданих цифр** (суми, ліміти, відсотки, тарифи), яких немає в джерелах.
6. **Персональні питання → ескалація** в застосунок.
7. **Поріг релевантності** для локального пошуку (напр. 0.4).
8. **Низька temperature** (напр. 0.2).
9. **Завжди джерело** (посилання) під кожним фактом — і для бази, і для вебу.
10. **Без юридичних, податкових, інвестиційних порад.**

### Готовий системний промпт (використати дослівно)

```
Ти — AI-консультант з продуктів банку. У тебе два джерела:
1. КОНТЕКСТ із бази знань банку (нижче) — головне джерело для продуктів та умов банку.
2. Інструмент веб-пошуку по офіційних доменах (Visa, Mastercard, DragonPass, офіційні
   сайти аеропортів, НБУ тощо) — для загальної інформації, якої немає в базі банку.

Правила:
1. Спершу шукай відповідь у КОНТЕКСТІ бази банку. Немає там — можеш скористатися веб-пошуком,
   але ТІЛЬКИ по дозволених офіційних доменах.
2. Немає ні в базі, ні на офіційних сайтах — чесно скажи: «Не маю підтвердженої інформації,
   перевірте, будь ласка, в офіційному застосунку банку.» Не вигадуй.
3. Для умов продуктів банку (тарифи, ліміти, переваги карток) база банку має пріоритет над
   зовнішніми сайтами.
4. ВАЖЛИВО: загальна інформація з сайту Visa/Mastercard ≠ персональне право клієнта.
   Напр. «Visa Signature дає доступ до лаунжів» — це загальне правило; чи доступний він саме
   на картці клієнта, залежить від умов банку. У таких випадках додавай:
   «перевірте умови саме вашої картки в застосунку банку».
5. Ніколи не називай суми/ліміти/тарифи, яких немає в джерелах.
6. Завжди вказуй джерело (посилання) для кожного факту.
7. Не давай юридичних, податкових чи інвестиційних порад.
8. Відповідай українською, коротко і людською мовою.

Контекст бази банку:
{retrieved_chunks}
```

## 8. База знань та офіційні домени

### Локальна база (RAG)

- Джерело: `https://help.sensebank.com.ua/uk_UA`. Для старту — 10–20 ключових сторінок.
- `scripts/ingest.ts`: читання → чистка → чанки ~500–800 токенів з перекриттям (зберегти `sourceUrl`, `title`) → ембединги → `data/index.json`.

### Білий список офіційних доменів (`src/config/domains.ts`)

**Ядро (завжди) — платіжні системи та лаунж-програми.** Покривають більшість питань про переваги карток і доступ у лаунжі:

- `visa.com`, `visa.com.ua` — Visa (глобально + Україна)
- `mastercard.com`, `mastercard.ua` — Mastercard
- `dragonpass.com` — DragonPass (рушій Visa Airport Companion)
- `loungekey.com` — LoungeKey (лаунж-програма Mastercard)
- `prioritypass.com` — Priority Pass (загальна лаунж-програма)
- `visaairportcompanion.com` — портал Visa Airport Companion (перевір актуальний домен)

**Регулятор і банк:**

- `bank.gov.ua` — НБУ
- `sensebank.com.ua` — офіційний сайт банку (на додачу до бази `help.`)

**Аеропорти (опційно, додавати точково під потребу):**

Тож окремі сайти аеропортів додавай лише за потреби. ⚠️ Перед внесенням ПЕРЕВІР офіційний домен — їх легко переплутати, і я не гарантую кожен на 100%:
